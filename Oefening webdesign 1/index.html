Vragenlijst
  1. TCP

    2. Transfer control protocol
    Het Transfer control protocol wordt gebruikt om data te versturen over netwerken/internet. Deze data wordt als individuele genummberde berichten verstuurd. Omdat data elke keer via een andere weg zijn bestemming zal bereiken, worden de genummerde berichten aan de andere kant weer in elkaar gezet op volgorde. Op deze wijze kan de ontvanger missende bestanden bij de verzender opvragen.

  2. ip-adres

    3. internet protocol adres
    Een internet protocol adres kan je vergelijken met een telefoonnummer van een computer. De Computer zoekt naar een website ( DNS naam ) in een DNS server. Uit deze DNS server krijgt hij een ip-adres terug en gebruikt dit "telefoon nummer" om contact te leggen met de gewenste site.

  3. DNS

    4. Domain Name System
    Het Domain Name System is een protocol om url's om te zetten naar ip adressen. De Domain Name System Server zal als hij een url binnen krijgt deze in een database matchen aan het juiste ip-adres.

  4. router

    5. Een router is een apparaat in een netwerk dat meerdere netwerken met elkaar verbind. De router werkt op laag 3 en maakt daarom gebruik van het ip protocol. Een router is een slim apparaat dat gebruikt maakt van een routing tabel om packets bij het juiste poortje naar buiten te sturen. Zo weet de router bijvoorbeeld het verschil tussen het home network en het internet en zal deze pakketten dus ook niet het internet opsturen.

  5. webserver

    6. Een webserver is een applicatie welke requests kan processen. De server ontvangt HTTP-requests en stuurt de opgevraagde documenten terug. Dit zijn meestal html documenten die in de browser worden getoond.

  6. HTML

    7. HyperText Markup Language
    HyperText Markup Language is een taal waarmee we websites bouwen. Deze taal maakt gebruik van tags om de site te structureren en formatten.

  7. HTML tag

    8. HyperText Markup Language tag
    Een HTML tag wordt gebruikt om de website structuur te geven. Zo kunnen wij met tags alineas maken, titles toevoegen of zoals je ziet ordered lists met tekst. Ook kunnen wij met tags omschrijvingen aan de site geven, op deze manier kan een zoekmachine zoals google ons koppelen aan zoektermen.


Hoe weet een browser wat voor soort bestand wordt geretourneerd


De browser ziet aan bijvoorbeeld de tag <HTML> dat het om een hypertext markup language gaat. En de tag <script src="...."> vertelt de browser dat er een javascript wordt uitgevoerd.


Doel van Keywords


Het doel van de keywords is om deze met een zoekmachine te kunnen doorzoeken. Zo kan bijvoorbeeld google jouw pagina aan een zoekopdracht koppelen.


Het doel van de description

De description is bedoeld om een webpagina samen te vatten in maximaal 160 characters. Deze description kan bijvoorbeeld door een zoekmachine worden getoont. Ook kan je deze zien als je met ctrl i de pagina info bekijkt.


Pagina broncode met ctrl + u

Als je in je browser ctrl + u indruk zal je een nieuw tabblad openen met daarin de broncode van de website welke je bezoekt.


Inspecting the elements

Waneer we in een browser ctrl + shift + i indrukken kunnen we elementen inspecteren. Zo zien wij meteen welk deel van de code verantwoordelijk is voor welk deel van de website. Ook kunnen wij hier aanpassingen maken welke meteen in de browser getoont worden.



Tags voor basis opmaak

In de basis opmaak van een html bestand kunnen wij op de volgende manier dingen veranderen:

  • Vet dit doen we met de tags <b> </b>
  • Cursiefdit doen we met de tags <i> </i>
  • Met nadruk dit doen we met de tags <em> </em>
  • onderstreept dit doen we met de tags <u> </u>
